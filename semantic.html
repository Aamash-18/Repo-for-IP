Semantic Controls:

In HTML4, cevelopers have to use their own id/class names to style elements: header, top, bottom, footer, menu, navigation, main, container, content, article, topnav, etc.
This is so difficult for search engines to identify the correct web page content. Now in HTML5 elements (header, footer, section etc.), this will become easier. It now allows data to be shared and reused.
Semantic elements can increase the accessibility of your website and helps to create a better website structure.

<article>

This element is used to defined article content within a document, page, application, or a website. It can be used to represent a forum post, a magazine, a newspaper article or a big story.

<section>

This element is used to represent the standalone section within an HTML document. A page can have various section and each section can contain, but headings for each section is not mandatory.
